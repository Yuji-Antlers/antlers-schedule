# ど素人向け 完全セットアップ手順書

このアプリを、**プログラミングの知識ゼロ・GitHubを触ったことがなくても**
動かせるように、画面の操作を1ステップずつ説明します。
上から順番に、そのとおりに進めれば完成します。

所要時間はだいたい **30〜40分**です。途中でわからなくなっても、
各ステップに「うまくいかないとき」を書いてあるので大丈夫です。

---

## はじめに：これから何をするのか（30秒で理解）

このアプリは「鹿島アントラーズの試合日程とチケット発売日を自動でチェックして、
新しい情報があればメールで知らせてくれる」ものです。

それを動かすために、**GitHub（ギットハブ）** という無料サービスを使います。
GitHubは「プログラムを置いておくと、決めた時間に自動で動かしてくれる場所」だと
思ってください。自分のパソコンを起動しっぱなしにする必要はありません。

やることは大きく5つです。

1. GitHubのアカウントを作る
2. プログラムの置き場所（リポジトリ）を作ってファイルを置く
3. メールを送る準備をする
4. パスワードなどの秘密情報を登録する
5. 自動で動く設定をオンにする

では始めましょう。

---

## 用語ミニ辞典（わからない言葉が出たら見る）

- **GitHub（ギットハブ）**：プログラムを保管して自動実行できる無料サービス。
- **リポジトリ**：プログラムの「置き場所」「フォルダ」のこと。略して「レポ」とも。
- **Actions（アクションズ）**：GitHubの「自動実行」機能の名前。
- **Secrets（シークレット）**：パスワードなどの秘密情報を安全にしまう金庫。
- **commit / push（コミット／プッシュ）**：ファイルをGitHubに保存すること。
  この手順書では画面の操作だけで済むので、意味はわからなくてOK。
- **ワークフロー**：「いつ・何を自動でやるか」を書いた設定ファイル。
  （すでに用意済みなので中身を書く必要はありません）

---

## ステップ0：必要なものを準備する

- **メールアドレス**（GitHub登録用）
- **通知を受け取りたいメールアドレス**（同じでもOK）
- パソコン（スマホでも一応できますが、PC推奨）

このプロジェクトのファイル一式（この手順書と同じフォルダにある以下）：

```
kashima_schedule.py
notify.py
tickets.py
README.md
.gitignore
.github/workflows/check.yml   ← .github というフォルダの中
```

> **注意**：`.gitignore` や `.github` のように **先頭にドット（.）が付くファイル/フォルダ**
> があります。これらは大事なので消さないでください。

## ステップ1：GitHubアカウントを作る（5分）

すでにアカウントを持っている人はステップ2へ。

1. ブラウザで **https://github.com/signup** を開く
2. 画面の指示に従って、メールアドレス・パスワード・ユーザー名を入力
   - **ユーザー名**は後でURLに使われます（例：`tanaka-kashima`）。
     あとから変えにくいので、シンプルなものにしておくと無難です。
3. 「Create account」を押す
4. 登録したメールアドレスに届く確認コードを入力
5. 無料プラン（Free）を選ぶ。有料の案内が出ても**無料でOK**です。

**うまくいかないとき**
- 確認メールが来ない → 迷惑メールフォルダを確認。それでも来なければ再送ボタン。

---

## ステップ2：リポジトリ（置き場所）を作る（3分）

1. GitHubにログインした状態で、画面右上の **「＋」マーク** をクリック
2. **「New repository」** を選ぶ
3. 設定を入力：
   - **Repository name（リポジトリ名）**：`kashima-schedule` など好きな名前
   - **Public（公開）か Private（非公開）か**：
     - **Public を選ぶのがおすすめ**です。理由は、GitHubの自動実行が
       Publicなら完全無料・無制限だからです。
     - 「日程やチケット情報が他人に見えても困らない」なら Public でOK
       （見られて困る個人情報は何も含まれません）。
     - どうしても非公開がよければ Private でも動きます（無料枠内）。
   - そのほかの項目（READMEを追加する等のチェックボックス）は
     **何も触らなくてOK**。
4. 緑色の **「Create repository」** ボタンを押す

これで空の置き場所ができました。

---

## ステップ3：ファイルをアップロードする（5分）

作ったリポジトリの画面に「uploading an existing file」というリンクが
あるはずです。なければ、リポジトリ画面の **「Add file」→「Upload files」** を押します。

### 3-1. ドット無しのファイルを上げる

1. 「Upload files」の画面で、`kashima_schedule.py`・`notify.py`・`tickets.py`・
   `README.md`・`SETUP_GUIDE.md` を、フォルダからドラッグ＆ドロップ
2. 画面下の緑ボタン **「Commit changes」** を押す（「保存」の意味）

### 3-2. `.gitignore` を上げる

同じく「Add file」→「Upload files」で `.gitignore` をドラッグ＆ドロップ →
「Commit changes」。

### 3-3. ワークフローファイルを上げる（ここだけ少しコツがある）

`.github/workflows/check.yml` は **フォルダの中に入ったファイル**です。
アップロード時にフォルダごと持っていく必要があります。

**いちばん簡単な方法**：
1. 「Add file」→ **「Create new file」** を選ぶ
2. 上部のファイル名の入力欄に、次を **そのまま** 打ち込む：
   ```
   .github/workflows/check.yml
   ```
   - `/`（スラッシュ）を打つと自動でフォルダ構造になります。
3. 大きな入力エリアに、`check.yml` の中身を **全部コピーして貼り付け**
   （手元の `check.yml` をテキストエディタで開いて全選択→コピー）
4. 「Commit changes」を押す

**うまくいかないとき**
- フォルダごとドラッグしてもOKな場合があります。ブラウザによります。
- ファイル名は1文字でも間違えると動きません。`.github/workflows/check.yml`
  を正確に。

これで全ファイルが置けました。リポジトリのトップ画面に
ファイル一覧が表示されていればOKです。

## ステップ4：メール送信の準備（10分）

通知メールを送るために、Gmailを使う方法を説明します
（いちばん手軽で、自分宛てに送るだけなら十分です）。

> Gmailアカウントを持っていない場合は https://accounts.google.com/signup で
> 作るか、後述の「Gmail以外を使う場合」を見てください。

### 4-1. Googleの2段階認証をオンにする

アプリパスワード（後述）を作るには、先に2段階認証が必要です。

1. https://myaccount.google.com/security を開く
2. 「2段階認証プロセス」を探してクリックし、画面に従ってオンにする
   （携帯番号でコードを受け取る設定が一般的）

### 4-2. アプリパスワードを作る

「アプリパスワード」は、このプログラム専用の16桁の合言葉です。
あなたの本当のGmailパスワードを使わずに済むので安全です。

1. https://myaccount.google.com/apppasswords を開く
   - もし「使用できません」と出たら、2段階認証がオンになっているか確認。
2. アプリ名を聞かれたら、わかりやすい名前（例：`kashima`）を入力
3. 「作成」を押すと、**16桁の文字列**（例：`abcd efgh ijkl mnop`）が出る
4. この16桁を**メモ**しておく（スペースは無視して `abcdefghijklmnop` でOK）
   - この画面を閉じると二度と表示されません。必ず控えてください。

## ステップ5：秘密情報（Secrets）を登録する（5分）

メールのパスワードなどを、GitHubの「金庫」に入れます。
プログラム本体には書かないので安全です。

1. 自分のリポジトリ画面の上のほうにある **「Settings」**（歯車・設定）をクリック
2. 左メニューの下のほうにある
   **「Secrets and variables」→「Actions」** をクリック
3. 緑色の **「New repository secret」** ボタンを押す
4. 「Name（名前）」と「Secret（値）」を入力して「Add secret」を押す。
   これを**項目の数だけ繰り返します**。

下の表のとおりに登録してください（Gmailを使う場合）。

| Name（この通りに正確に） | Secret（あなたの値） |
|---|---|
| `MAIL_METHOD` | `smtp` |
| `MAIL_TO` | 通知を受け取りたいメールアドレス |
| `MAIL_FROM` | あなたのGmailアドレス |
| `SMTP_HOST` | `smtp.gmail.com` |
| `SMTP_PORT` | `587` |
| `SMTP_USER` | あなたのGmailアドレス |
| `SMTP_PASS` | ステップ4で作った16桁のアプリパスワード |

> **ポイント**：Nameは表のとおり**大文字・記号まで正確に**。
> 1文字でも違うと動きません。値の前後によけいなスペースを入れないこと。

登録が終わると、名前の一覧が表示されます（値は二度と見られませんが、
それで正常です）。間違えたら、その項目を消してもう一度作り直せます。

---

## ステップ6：自動実行の権限をオンにする（2分）

プログラムが結果を保存できるように、書き込みの許可を出します。

1. リポジトリの **「Settings」** を開く
2. 左メニューの **「Actions」→「General」** をクリック
3. 下のほうにスクロールし、**「Workflow permissions」** の項目を探す
4. **「Read and write permissions」** を選ぶ（点を入れる）
5. **「Save」** を押す

**この設定を忘れると、後で保存のところでエラーになります。**忘れずに。

---

## ステップ7：Webページ公開の設定（3分）

最新の日程一覧をネット上で見られるようにします（任意ですが簡単です）。

1. リポジトリの **「Settings」** を開く
2. 左メニューの **「Pages」** をクリック
3. **「Build and deployment」** の **「Source」** という欄を
   **「GitHub Actions」** に変更する
   - プルダウンから選ぶだけ。保存ボタンは不要なことが多いです。

これで、後でプログラムが動くと
`https://（あなたのユーザー名）.github.io/（リポジトリ名）/`
で日程表が見られるようになります。

---

## ステップ8：はじめて動かしてみる（3分）

設定がそろったので、手動でテスト実行します。

1. リポジトリの上のメニューの **「Actions」** をクリック
2. （初回は「ワークフローを有効にするか」聞かれることがあります。
   「I understand my workflows, go ahead and enable them」を押す）
3. 左側に **「鹿島日程チェック」** という名前が見えるのでクリック
4. 右側の **「Run workflow」** ボタンを押し、出てきた小窓でもう一度
   **「Run workflow」** を押す
5. 少し待つと一覧に実行中の行が出ます。黄色い丸（実行中）→
   **緑のチェック（成功）** になれば成功です。

初回は全部の試合が「新しい情報」とみなされるので、**登録したメールアドレスに
通知メールが届きます**。届いたら大成功です。

> ※今が試合のオフ期間だと、取得できる試合が少なかったり、チケット情報が
> 空のことがあります。その場合でもエラーが出ていなければ正常です。

**うまくいかないとき（赤いバツが出たとき）**
- 赤いバツの行をクリックすると、どこで失敗したか赤字で表示されます。
- よくある原因：
  - ステップ6の「書き込み権限」を忘れている → 設定し直して再実行
  - Secretsの名前が表と違う／スペルミス → 見直して直す
  - アプリパスワードの打ち間違い → 作り直して `SMTP_PASS` を更新
- 直したら、またステップ8の手順で「Run workflow」を押せば再挑戦できます。

## ステップ9：あとは放っておくだけ

ここまでできたら、もう何もしなくて大丈夫です。
プログラムは **毎日 朝8時と夜20時（日本時間）に自動で動き**、
新しい情報があったときだけメールが届きます。

何も変化がない日はメールは来ません（それが正常です）。

## よくある質問

**Q. お金はかかりますか？**
A. かかりません。Publicリポジトリなら自動実行は無制限無料、
   Gmailもメールも無料の範囲です。

**Q. メールが多すぎたら？**
A. 変化があったときだけ届くので、通常は多くありません。
   実行の回数を減らしたい場合は、`.github/workflows/check.yml` の
   時間設定（cron）の行を減らせます（READMEに説明あり）。

**Q. 通知が来るタイミングはぴったり正確？**
A. GitHubの混み具合で、設定時刻から数分〜十数分ずれることがあります。
   「発売日を見逃さない」用途には十分ですが、秒単位の速さはありません。

**Q. しばらく使わないと止まると聞いたけど？**
A. 60日間まったく更新がないと自動実行が一時停止します。
   普通に毎日動いていれば該当しません。もし止まったら、
   Actions画面から一度手動実行すれば復活します。

**Q. Gmail以外を使いたい**
A. 「Resend」というサービスも使えます。その場合、Secretsを
   `MAIL_METHOD` = `resend`、`RESEND_API_KEY` = （Resendで取得した鍵）、
   `MAIL_TO`・`MAIL_FROM` を登録します（SMTP系の項目は不要）。
   詳しくは README.md を参照。

**Q. 取得元のサイトが変わって動かなくなったら？**
A. 実行結果が「解析できた試合数: 0」になっていたら、サイトの作りが
   変わった可能性があります。その場合は調整が必要なので、
   実行ログ（Actions画面で確認できます）を持って相談してください。

## 困ったときのチェックリスト

うまくいかないときは、上から順に確認してください。

1. Secretsの名前は表と**完全に一致**しているか（大文字・記号も）
2. ステップ6の**書き込み権限**をオンにしたか
3. アプリパスワードは**正しく16桁**入れたか（スペース無しでOK）
4. ワークフローファイルは `.github/workflows/check.yml` という
   **正しい場所と名前**に置かれているか
5. 通知先メール（`MAIL_TO`）のアドレスは正しいか・迷惑メールに入ってないか

それでも解決しないときは、Actions画面の失敗した実行を開いて、
**赤字のエラーメッセージ**を控えて相談すると早く解決できます。
